# 좌석 추천 + 예매 방법 시각 안내 + 웹 검색 기능 설계

## 배경

현재 챗봇(`backend/main.py`)은 매 질문을 독립적으로 처리하는 단일 턴 Q&A이며, 시스템 프롬프트에 박힌 고정 사실(예매처, 예매 시작 시점, 홈구장 등)에만 근거해 텍스트로만 답변한다. 이번 변경의 목표는 다음 세 가지를 추가하는 것이다.

1. 사용자 맞춤 좌석 추천 (구장 지도 시각 자료 포함)
2. 예매 방법 단계별 안내 (체크리스트 시각 자료 포함)
3. 시스템 프롬프트에 없는 질문(게이트, 주차 등)에 대한 실시간 웹 검색 기반 답변

## 아키텍처

### 멀티턴 대화 지원
- 프론트엔드(`script.js`)가 대화 기록(`{role, content}` 배열)을 메모리에 유지하고, 매 요청마다 전체 기록을 백엔드로 전송한다.
- 새로고침하면 기록 초기화 (DB/세션 저장 없음 — 테스트 용도 프로젝트 규모에 맞춤).
- 백엔드는 받은 기록 전체 + 시스템 프롬프트를 OpenAI에 전달해 맥락 유지.

### 구조화 응답
백엔드는 OpenAI Structured Outputs(JSON 스키마)를 사용해 다음 형태로 응답한다.

```json
{
  "answer": "내야석을 추천드려요! 시야가 좋고...",
  "visual": { "type": "seat_zone", "zone": "infield" }
}
```

- `visual.type`: `"none" | "seat_zone" | "booking_steps"`
- `visual.zone` (seat_zone일 때만): `"infield" | "outfield" | "cheer"`
- 기존 응답의 `link` 필드(인터파크 URL)는 그대로 유지.
- 그림(SVG/체크리스트) 마크업 자체는 LLM이 생성하지 않고 프론트엔드에 고정 템플릿으로 둔다 — `zone`/`type` 값만 LLM이 결정. 안전성과 일관된 디자인을 위함.

### 웹 검색 도구
- OpenAI 웹 검색 도구(web search tool)를 모델 호출에 함께 제공.
- 시스템 프롬프트에 있는 고정 사실로 답변 가능한 질문은 검색 없이 즉답.
- 시스템 프롬프트에 없는 질문(주차, 매장 위치, 날씨로 인한 경기 취소 여부 등)은 모델이 자율적으로 웹 검색을 실행한 후 답변.
- 검색 기반 답변은 사용자가 신뢰도를 판단할 수 있도록 출처/검색 기반임을 답변에 표시.
- 트레이드오프: 검색이 실행되는 케이스는 응답 지연(수 초) 및 호출 비용이 늘어남. 테스트 용도 프로젝트라 허용.

## 시스템 프롬프트 사실 확장 (게이트 정보)

웹 검색(2026-06-24 수행)으로 확인한 팬 블로그/후기 기반 정보(공식 출처 아님, 참고용)를 고정 사실로 추가:

- 지하철 종합운동장역 5번 출구(1루 방향이 더 빠름) / 6번 출구(3루·모바일티켓 방향이 더 빠름)
- 1루 게이트는 역에서 왼쪽 방향, 3루 게이트는 오른쪽 방향
- 외야 출입구는 내야 게이트보다 더 바깥쪽(중앙매표소 기준 3루외야가 더 안쪽, 1루외야가 더 바깥쪽)
- 내야 출입구는 총 4개, 좌석 블록별로 입장 게이트가 다름
- LG 홈경기 기준 1루 = LG 팬 자리, 3루 = 원정팀 자리
- 정확한 게이트 번호까지는 공식 출처로 확인 못 했으므로, 챗봇은 이 정보를 "대략적인 방향 안내"로만 제공하고 정확한 번호는 티켓 QR/현장 안내판 확인을 권장하도록 안내

## 좌석 추천 대화 흐름

1. 사용자가 "좌석 추천해줘" 같은 질문을 하면, 시스템 프롬프트 지침에 따라 모델이 예산/동행 인원/응원 성향 등을 먼저 되묻는다 (`visual: none`).
2. 정보가 충분히 모이면 모델이 구역(`infield`/`outfield`/`cheer`) 중 하나를 추천하며 `visual: { type: "seat_zone", zone: ... }`로 응답.
3. 정보가 부족하면 계속 추가 질문만 하고 `visual: none` 유지.

## 프론트엔드 렌더링

### 구장 지도 SVG (seat_zone)
- 경기장을 위에서 본 형태의 SVG: 외야석(좌/우), 내야석(1루/3루), 응원석 영역을 색상 블록으로 표시.
- `zone` 값에 해당하는 영역만 테두리 강조(굵은 테두리 + 진한 색)로 하이라이트.
- 답변 텍스트 bubble 아래에 삽입.

### 예매 단계 체크리스트 (booking_steps)
- 고정 4단계: ① 인터파크 티켓 접속 → ② 날짜·홈경기 확인 → ③ 좌석 선택 → ④ 결제 및 전자티켓 수령.
- 번호 카드로 세로 나열, 답변 텍스트 bubble 아래에 삽입.
- 기존 "🎫 인터파크 티켓에서 예매하기" 링크 버튼은 그대로 유지.

두 시각 자료 모두 순수 HTML/CSS/SVG (외부 이미지·라이브러리 의존 없음)로, 기존 `index.html` 스타일(파스텔 컬러, 8px 라운드 코너)에 맞춰 제작한다.

## 에러 처리

- OpenAI 호출 자체가 실패하면(네트워크/키 오류 등) 기존과 동일하게 고정 안내 문구로 폴백하고 `visual: none`.
- 모델이 스키마에 없는 `zone`/`type` 값을 주는 경우는 Structured Outputs의 enum 제약으로 원천 차단됨.

## 테스트 (수동)

1. "좌석 추천해줘" → 예산/인원 등 질문을 1~2회 받은 뒤 → 구장 지도 SVG와 추천 구역 강조 표시 확인
2. "예매 어떻게 해?" → 체크리스트 표시 확인
3. 시스템 프롬프트에 없는 질문(예: "주차장 있어?") → 웹 검색이 실행되고 답변에 반영되는지 확인
4. 일반 잡담/무관한 질문 → 시각 자료 없이 텍스트만 응답하는지 확인
